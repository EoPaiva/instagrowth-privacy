InstaGrowth IA uPaiva

Política de Privacidade

Última atualização: 16 de maio de 2026

A extensão InstaGrowth IA uPaiva foi projetada com privacidade como prioridade. Este documento explica o que coletamos, o que armazenamos, o que enviamos para servidores externos e o que NÃO fazemos.

TL;DR

  • Quase tudo fica no seu navegador (chrome.storage.local).
  • A extensão nunca pede sua senha do Instagram.
  • Sua chave de IA NUNCA é enviada para nossos servidores.
  • Apenas dados de pagamento PIX e a license key são enviados ao backend da uPaiva.
  • Você pode apagar todos os dados a qualquer momento em Configurações → "Limpar todos os dados".

O que a extensão coleta e armazena

Tudo abaixo fica apenas no seu navegador, em chrome.storage.local. Nada é enviado para nossos servidores sem que esteja descrito na seção "Comunicação com servidores externos" abaixo.

Dados de uso da extensão

  • Perfis capturados (username, nome, foto, bio, contagens, classificação heurística, origem da captura)
  • Histórico de quem você seguiu pela extensão (username, data, origem, status de retorno de follow)
  • Fila atual e fila pendente (perfis na fila de crescimento)
  • Configurações: limites, delays, preset escolhido
  • Nicho configurado (nicho, público-alvo, objetivo, tom, região)
  • Hashtags testadas
  • Histórico do Prompt Studio (últimos 50 prompts gerados)
  • Favoritos do Prompt Studio
  • Status do termo de aceite do Auto Pleno (versão + timestamp + device ID + hash do conteúdo)

Dados de Premium

  • Device ID aleatório (UUID gerado localmente para identificar este navegador)
  • E-mail usado na compra
  • License key (criptografada em trânsito)
  • Plano ativo e data de expiração
  • Features liberadas
  • Timestamp da última verificação

Chave de IA (BYOK)

  • Provedor escolhido (Claude / OpenAI / Gemini)
  • Chave de API do provedor — armazenada localmente com obfuscação XOR + base64 (não é criptografia forte; recomendamos restringir a chave por uso/IP no painel do provedor)

O que a extensão NÃO coleta

  • Senha do Instagram — nunca pedimos.
  • Mensagens diretas (DMs) — não acessamos.
  • E-mail ou dados pessoais de outros perfis — apenas dados públicos.
  • Histórico de navegação fora do instagram.com — a extensão só roda em https://www.instagram.com/*.
  • Dados sensíveis (CPF, cartões, etc.) — apenas e-mail no fluxo de PIX.

Comunicação com servidores externos

1. Instagram (instagram.com)

A extensão faz chamadas à API interna pública do Instagram, no contexto da sua sessão logada (com seus cookies). Especificamente:

  • GET /api/v1/users/{id}/info/ — detecta seu username logado
  • GET /api/v1/users/web_profile_info/?username=... — info de perfis
  • GET /api/v1/friendships/{id}/followers/ — lista de seguidores
  • GET /api/v1/friendships/{id}/following/ — lista de seguindo
  • POST /api/v1/friendships/create/{id}/ — segue um usuário (quando você usa a Fila)
  • POST /api/v1/friendships/destroy/{id}/ — deixa de seguir (se você usar)
  • GET /api/v1/media/{id}/likers/, /comments/ — curtidores/comentadores (experimental)
  • POST /api/v1/feed/tag/{tag}/sections/ — hashtags (experimental)

Importante: estas requisições são feitas no seu navegador, com seus cookies, exatamente como o site oficial do Instagram faria. Não enviamos nada disso para a uPaiva.

2. Backend uPaiva (Railway)

URL: https://beautiful-laughter-production-ba79.up.railway.app

A extensão se comunica com o backend uPaiva apenas para:

  • Gerar PIX de compra (envia: e-mail, plano, device ID, app ID)
  • Verificar status do PIX (appId=instagrowth)
  • Ativar license key (envia: e-mail, key, device ID, app ID)
  • Validar status da licença a cada 24h
  • Recuperar acesso por código de 6 dígitos
  • Listar planos disponíveis
  • (Auto Pleno) Registrar aceite do termo para auditoria legal

O backend nunca recebe: sua chave de IA, perfis capturados, fila, histórico de follows, prompts gerados.

3. Provedor de IA (Anthropic / OpenAI / Google)

Se você configurar o Growth Coach IA ou o Prompt Studio com sua própria chave (BYOK), a extensão envia diretamente para o provedor que você escolheu:

  • https://api.anthropic.com/v1/messages (Claude)
  • https://api.openai.com/v1/chat/completions (OpenAI)
  • https://generativelanguage.googleapis.com/v1beta/... (Gemini)

Cada requisição contém: prompt, contexto do nicho (se você configurou), e a chave que você forneceu. Nada disso passa pelos servidores da uPaiva.

Cookies

A extensão lê alguns cookies do Instagram apenas no contexto do instagram.com:

  • ds_user_id — para identificar seu ID de usuário logado
  • csrftoken — para enviar nos headers das requisições

Esses cookies são lidos no seu navegador e usados nas requisições à API do Instagram. Não enviamos cookies para nossos servidores.

Termo de Aceite do Auto Pleno

Quando você habilita o modo Auto Pleno, registramos localmente (e enviamos ao backend uPaiva para auditoria legal):

  • Timestamp do aceite
  • Device ID
  • Versão do termo aceito
  • Hash SHA-256 do texto integral do termo
  • User-Agent

Isso é uma proteção legal mútua: você comprova que entendeu os riscos, e a uPaiva comprova que avisou.

Seus direitos

  • Apagar todos os dados: Configurações → "Limpar todos os dados". Apaga tudo do chrome.storage.local e desativa Premium neste navegador.
  • Apagar dados específicos: você pode remover perfis capturados, itens da fila ou do histórico individualmente.
  • Solicitar exclusão de dados do backend: envie e-mail para o endereço informado em upaiva.dev. Removemos seu device ID, e-mail de compra e registro de aceite do termo (sem afetar registros contábeis exigidos por lei).

Não somos afiliados ao Instagram

A InstaGrowth IA uPaiva não é afiliada, endossada ou patrocinada pela Meta Platforms, Inc. ou pelo Instagram. Esta extensão é fornecida "como está" (as-is), sem garantias.
